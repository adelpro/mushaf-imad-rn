# Contributing to Mushaf Imad

First off, thank you for considering contributing to Mushaf Imad! It's people like you that make this app better for everyone.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. Please be respectful and professional in all interactions.

## How Can I Contribute?

### Reporting Bugs

- **Check if the bug has already been reported** by searching the [Issues](https://github.com/adelpro/mushaf-imad-rn/issues) page.
- If you can't find an open issue that describes the problem, [open a new one](https://github.com/adelpro/mushaf-imad-rn/issues/new). Use our **Bug Report** template to provide as much detail as possible.

### Suggesting Enhancements

- **Check if the enhancement has already been suggested**.
- If not, [open a new issue](https://github.com/adelpro/mushaf-imad-rn/issues/new) using the **Feature Request** template.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **Ensure the test suite passes**.
4. **Make sure your code lints**.
5. **Update documentation** if necessary.
6. **Open a Pull Request** with a clear title and description.

## Style Guide

- We use **Clean Code** principles.
- Use **TypeScript** for all logic.
- Follow the existing formatting (Prettier/ESLint rules).
- Keep components focused and reusable.

## Translations

Mushaf Imad supports multiple languages. If you'd like to contribute a new translation or improve an existing one, please check the `src/i18n` directory.

Thank you for your contributions!
